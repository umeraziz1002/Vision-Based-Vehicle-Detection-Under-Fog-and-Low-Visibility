Build a complete Flutter mobile application for a Final Year Project titled:
"Vision-Based Vehicle Detection under Fog and Low Visibility Conditions"

The app is the mobile companion to an existing Flask web backend with REST API
endpoints already deployed. The UI must be pixel-perfect consistent with the
existing web app design system described below.

API INTEGRATION
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Base URL: configurable in api_constants.dart (default http://10.0.2.2:5000
for Android emulator, http://localhost:5000 for iOS simulator)

Endpoints:
  GET  /api/v1/health              → health check on app start
  POST /api/v1/detect/image        → multipart/form-data: file, confidence, iou
  POST /api/v1/detect/video        → multipart/form-data: file, confidence, iou
  GET  /api/v1/stats               → dashboard statistics
  GET  /api/v1/history?limit=20    → recent detections list

Use the http package (not dio) for simplicity.
Show SnackBar on API errors with danger color background.
Show loading overlay (full-screen) during detection requests.

DetectionResult model:
  success, fileType, originalFile, outputFile, outputUrl, results{
    totalDetections, detections[{class, confidence, bbox}],
    classCounts{}, inferenceTime, imageSize{width,height}
  }

HistoryItem model:
  fileType, originalFile, outputFile, totalDetections,
  classCounts{}, timestamp, source

ANIMATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

- FadeSlideAnimation widget: wraps any child, slides up 20px + fades in
  on mount using AnimationController + CurvedAnimation (Curves.easeOut, 400ms)
  with optional delay parameter for staggered entrance
- Stat numbers animate from 0 to target value on screen entry
  using Tween<int> + AnimationController (600ms, Curves.easeOut)
- Detection bounding box demo on home screen: 3 boxes pulse opacity
  using AnimationController with repeat(reverse: true)
- Scan line on home demo: slides top→bottom using AnimationController
  with repeat, drawn via CustomPainter
- Progress bar on loading overlay: animated shimmer using gradient
  and AnimationController

1. ThemeData must set:
   - scaffoldBackgroundColor: bgPrimary
   - colorScheme: dark scheme with primary=accent, surface=bgSecondary
   - All default widget colors overridden to match the design system
   - No default blue anywhere

2. Settings screen (accessible from About via gear icon):
   - Text field to configure the Flask server base URL
   - Saved to SharedPreferences
   - "Test Connection" button that calls /api/v1/health

3. File picker must support:
   - Images: jpg, jpeg, png, bmp, webp
   - Videos: mp4, avi, mov, mkv
   - Show file size and type after selection
   - Validate max 100MB before uploading

4. Result image display:
   - Load from server URL: baseUrl + outputUrl
   - Use CachedNetworkImage with shimmer placeholder
   - Pinch-to-zoom using InteractiveViewer

5. Error handling:
   - Network timeout: 120 seconds (videos can be slow)
   - Show retry button on failure
   - Graceful empty states on dashboard when no history

6. Generate ALL files completely — no placeholders, no TODOs.
   Every screen, widget, model, and service must be fully implemented
   and ready to run with flutter run.
